How I Got Here

I’ve always been in awe of computers and how quickly they've changed our experience on this planet forever. My first exposure to programming beyond HTML was when I was 13, leafing through a C++ book wondering what kind of crazy you had to be to enter some complex phrases into a text editor that a computer could read and bundle into a real, working, useful program. It seemed like magic without the illusion.

Many years later in 2014, I randomly found a link on Reddit for Harvard’s CS50 open course-ware that people were gushing about. It seemed very gimmicky, but I gave it a try. After watching David Malan tear a phone book in half as a human Strongman algorithm, and working through the Scratch problem set, the course immediately jumped into C, with gedit. C is for Cake right? I clawed my way through the first C problem set for many hours trying print a mario-themed half-pyramid of hashes as tall as the user wants. After much frustration, and breaking the problem apart, and breaking each part apart, the process started to take a linear shape. After the first few problem sets, I was hooked. By the start of 2015, I read about Free Code Camp.

After a few months of Free Code Camp challenges, I was all in. For the next year and beyond, I learned to build responsive UI's with HTML5 and SASS, add interactivity with JavaScript, make HTTP requests from clients to powerful 3rd party APIs, and build my own APIs on servers running NodeJS. JavaScript, especially with NodeJS, still feels very much like the Wild West, but it's very flexible and performant for a wide range of web applications, tooling, and things. Plus, the communities surrounding the language, libraries, and frameworks live for helping coders build great things, and helping forward-thinking companies change lives. It's getting better every year.
